Git

  • A branch allows a user to work on a "copied" version of the Main file without affecting the Main file itself or other users
  • git status check what branch we're currently on
  • git checkout -b branch name: creates a new branch and switches to it
  • checkout command == move working branch into new branch
  • -b == creates a new branch
  • feature/desired-branch-name == name of new branch
  • add == adds modifications in the current working branch to staging area
  • -A == will add all changes
  • git commit == commits files to be changed
  • -m "your desired message" == adding a comment of current staged changes
  • pull == receives a branches changes to local environment
  • origin == the source of the pull in the GitHub repo
  • push == push local changes to remote branch and can create a new remote branch or add new changes to existing branch

Git flow process

  1. Create a local feature branch
  2. check status
  3. create feature branch by following:
    git checkout -b feature/branch-name(ex: starter code)
  4. Add starter code to current working branch: git add -A
  5. Previous step staged the file to be commited
  6. Commit the changes & add a commit message:
    git commit -m "desrcibe changes"
  7. Before pushing up to repo, double-check that local and GitHub branch are in sync:
    git pull origin main
  8. Branches are now in sync!
  9. push changes to remote branch: git push origin feature/feature-name
  10. Go to GitHub repo in browser and select the proper feature branch then hit
    "compare & pull request"
  11. Give PR a descriptive title
  12. In the PR body: add instructions for team members reviewing the PR about what changes were made, how the feature is supposed to function, and how to start the feature so it can be tested
  13. designate the base branch as main and the comparison branch as feature/branch-name
  14. select "Create Pull Request"
